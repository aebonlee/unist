## 2. 기술 스택

- **Frontend**: React 19 + TypeScript + Vite 7, React Router 7 (SPA)
- **인증/DB**: Supabase (`@supabase/supabase-js`) — Google·Kakao OAuth + 이메일
- **콘텐츠**: react-markdown + remark-gfm + rehype-raw (강의안 마크다운 렌더링)
- **디자인**: dasco/kdn 공용 디자인 시스템 — 다크 블루(navy `#1B2A4A`) 기본 + 5색 테마
- **이미지**: sharp (OG 이미지 생성)
- **배포**: GitHub Pages (gh-pages 브랜치) + 커스텀 도메인
- **빌드 환경**: Node v22.22.2 (nvm)

## 4. 개발 작업 이력

### (1) 초기 구축
- dasco 템플릿을 클론해 data 저장소에 이식, 레거시 파일 정리.
- 브랜딩 전환: id `data`, name `Data Lab`, dbPrefix `data_`, CNAME/robots/sitemap/OG meta → data 도메인.
- 기본 다크 블루 + 5색 테마(blue/red/green/purple/orange) 유지.
- KERIS 커리큘럼 반영: Home, Curriculum, 모듈1~3 강의안 작성.

### (2) 인증 & DB
- `.env`(VITE_ 접두사)에 Supabase URL/anon key. service-role/PAT는 클라이언트 제외.
- `user_profiles`(RLS+가입 트리거), `data_settings`(실습실/배당 RLS) SQL 작성·적용.
- Google·Kakao OAuth는 `src/utils/auth.ts`에 배선.

### (3) 배포
- 빈 페이지 이슈: Pages가 main 루트(`/src/main.tsx`) 서빙 → 빌드 산출물 미사용.
- 해결: `dist` → `gh-pages` 브랜치 배포(`npm run deploy`) + Pages 소스 전환(API).
- OG 이미지(다크블루·막대차트 모티프) sharp로 생성.

### (4) AI 메뉴 포팅 (ai-free → data)
- learn/prompt/tools/examples/playground/admin 페이지 + aiProviders/aiClient/tokenAllocation/examples 이식.
- `aifree.css` 추가 + index.css import (스타일 적용).
- GuidePage에 옵션 `heroEyebrow` 헤더 지원.

### (5) UI 다듬기
- 메뉴 "과정 소개"→"About", nav gap 36→28px, 강의안 드롭다운 라벨 간소화, 드롭다운 폭 자동(max-content).

### (6) 모바일 최적화 (Playwright 390/360px 실측)
- GuidePage 모바일 사이드바 세로 스택(@900), 표는 컨테이너 내 가로 스크롤.
- aifree.css `@media` 순서 버그 수정(base 뒤로 이동) + 600px 보강.
- 햄버거 드롭다운 전체 폭 복원.
- About/CompanyIntro 인라인 고정 그리드 → className+미디어쿼리 반응형화.
- **결과: 전 페이지 가로 오버플로우 0px 검증 완료.**
